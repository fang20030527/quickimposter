# Quick Imposter SVG 图标替换设计

## 目标

将用户截图红框内的图标与右下双手机插画重绘为可复用的矢量 SVG，并替换到工作区 `design.pen` 的桌面端与移动端原型中。替换后保持现有卡片、文案、间距和页面布局不变。

## 交付物

- 3 个卖点图标：闪电、手机、盾牌。
- 3 个玩法步骤图标：玩家群组、手机窥词、对话投票。
- 2 个步骤连接箭头图标。
- 1 张完整双手机传递插画，包含锁、眼睛、旋转箭头、星芒和紫色强调线。
- 更新后的 `design.pen`。
- 桌面端与移动端 PNG 预览图。

## 视觉规范

- 延续现有品牌的深紫、亮紫和黄色配色，以及圆润几何轮廓、深紫描边和适度渐变。
- SVG 必须使用明确的 `viewBox`，在 Pencil 中按原始宽高比等比例缩放，不拉伸、不裁切。
- 小图标保持视觉粗细一致，在白色卡片上具有足够对比度。
- 双手机插画完整重绘为一张 SVG，保留前后叠放关系、透视感和装饰元素。
- 双手机插画中的 `Press and hold`、`Your word appears here`、`PASS CLOCKWISE` 等文字使用 `Helvetica Neue`，并提供 `Helvetica`、`Arial`、`sans-serif` 回退字体。

## 文件与节点结构

- SVG 素材保存到项目内独立资源目录，使用语义化英文文件名。
- 小图标各自保留为独立 SVG；双手机插画作为一张组合 SVG。
- `design.pen` 中只替换对应图标或插画节点，不合并卡片文字和背景。
- 桌面端和移动端复用同一组 SVG 素材，并分别保持各自当前尺寸与对齐方式。

## 实施顺序

1. 定位 `design.pen` 中桌面端与移动端的相关图标节点及当前尺寸。
2. 绘制并验证 8 个小图标 SVG 与 1 张双手机插画 SVG。
3. 将现有图标节点替换为 SVG 图像节点，保持容器尺寸和布局参数不变。
4. 确认双手机插画使用等比例适配，且插画内文字使用 Helvetica 字体家族。
5. 导出桌面端与移动端预览，检查清晰度、比例、裁切、对齐和字体显示。

## 验收标准

- 红框中的所有图标和双手机插画均来自新绘制的 SVG，不再依赖 Emoji 或位图占位图标。
- 图标风格、配色和视觉重量与参考截图一致。
- 桌面端和移动端均无拉伸、裁切、错位或布局变化。
- 双手机插画内文字使用 Helvetica 字体家族，内容清晰可读。
- SVG 文件可独立打开，Pencil 预览可正常渲染，导出的 PNG 无缺失元素。

## 范围限制

- 不修改红框之外的 Logo、标题艺术字、设置表单、FAQ、页脚或网页代码。
- 不改变现有英文文案和页面信息架构。
- 本次只更新静态 Pencil 原型及其设计资源，不实现交互。
